CONTEXTO DEL PROYECTO
----------------------
Aplicación de tienda en línea construida con:
  - Backend:   FastAPI + SQLAlchemy (ORM)
  - Base de datos: PostgreSQL (migrado desde SQLite durante esta sesión)
  - Frontend:  Jinja2 Templates + CSS puro (sin frameworks)
  - Auth:      Sesiones en memoria con passlib bcrypt

El proyecto tiene 3 módulos principales:
  1. Usuarios (users) — CRUD completo
  2. Productos (products) — CRUD completo
  3. Valoraciones (ratings) — Create + Read

BLOQUE 1 — MIGRACIÓN SQLite → PostgreSQL
================================================================================

Archivos modificados: database.py, requirements.txt

- Se reemplazó la URL de conexión SQLite por PostgreSQL:
    postgresql://postgres:Superduper69@localhost:5432/UpSkillss_db?client_encoding=utf8
- Se eliminó connect_args={"check_same_thread": False} (exclusivo de SQLite)
- Se instaló y agregó psycopg2-binary como driver de PostgreSQL
- Se añadió ?client_encoding=utf8 para resolver UnicodeDecodeError en Windows
  (problema conocido: PostgreSQL devuelve errores en español con encoding Latin-1

NOTAS TÉCNICAS IMPORTANTES
================================================================================

SESIONES:
  Las sesiones se almacenan en un dict en memoria (sessions = {}). Al reiniciar
  el servidor todos los usuarios quedan deslogueados. Esto es intencional para
  el alcance del proyecto (demo). Para producción se requeriría JWT o sesiones
  en DB.

IMÁGENES:
  Se guardan en static/images/ con nombre uuid4_original.ext
  El campo product.image puede contener 1 o 3 nombres separados por coma.
  Al eliminar un producto se eliminan sus archivos de imagen del filesystem.
  Al eliminar una cuenta, las imágenes de sus productos NO se eliminan
  del filesystem (solo de la DB). Para producción convendría limpiarlas.

MIGRACIONES:
  Este proyecto no usa Alembic. Los cambios de esquema requieren ALTER TABLE
  manual en pgAdmin. Columna agregada durante esta sesión:
    ALTER TABLE products ADD COLUMN link VARCHAR;

PALETA DE COLORES (variables CSS en styles.css):

BLOQUE 12 — LIMPIEZA, SEGURIDAD Y BUENAS PRÁCTICAS
================================================================================

Archivos eliminados: mi_tienda.db (SQLite huérfano), schemas.py (Pydantic sin uso)

CREDENCIALES EN .ENV:

- database.py ahora usa python-dotenv: load_dotenv() + os.getenv("DATABASE_URL")

HELPERS DRY (dependencies.py):
- build_ratings_list(): extrae lógica duplicada en 3 rutas → 1 función
- paginate(): extrae lógica duplicada en 2 rutas → 1 función

SEGURIDAD — FILENAMES:
- Antes: f"{uuid.uuid4()}_{img.filename}" → nombre original podía tener
  comas (rompía el split), espacios, o path traversal
- Ahora: ext = os.path.splitext(img.filename)[1].lower()
         filename = f"{uuid.uuid4()}{ext}"  ← solo UUID + extensión

SEGURIDAD — VALIDACIÓN DE TIPO DE ARCHIVO:
- Función is_valid_image(file) lee los primeros 12 bytes (magic bytes)
- Firmas aceptadas: JPEG (ff d8 ff), PNG (89 PNG), GIF, WebP (RIFF), BMP
- Aplicada antes de guardar en add_product y edit_product
- Error mostrado al usuario si el archivo no es imagen válida

SEGURIDAD — CONTRASEÑAS:
- Validación mínimo 6 caracteres en registro (antes sin límite)

CONSISTENCIA — WHATSAPP:
- En auth.py: whatsapp: str = Form("") → ahora opcional (era Form(...) requerido)
  Alineado con models.py donde whatsapp = Column(String, nullable=True)

RENDIMIENTO — N+1 QUERIES:
- Antes: por cada producto listado SQLAlchemy hacía query separada para ratings
  (12 productos = 13 queries)
- Ahora: joinedload(models.Product.ratings) carga todo en 1 sola query
- Aplicado en: read_products, user_detail, my_products

PAGINACIÓN — /user/{id}:
- Antes: mostraba todos los productos del vendedor sin límite
- Ahora: acepta ?page=N, usa PAGE_SIZE = 12, muestra controles de paginación
- Estilos de pagination/.page-btn añadidos a user_detail.css

HANDLER 500:
- Añadido @app.exception_handler(500) en main.py
- Creado templates/500.html (reutiliza notfound.css)

IMÁGENES (nota actualizada):
- Los nombres de archivo ya no incluyen el nombre original del usuario
- Solo UUID + extensión: evita comas en filenames que rompían el split
